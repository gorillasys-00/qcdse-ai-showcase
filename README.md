# エンタープライズ品質を実装した建設業向け次世代AI施工管理SaaS「QCDSE AI」

## プロジェクト概要

「QCDSE AI」は、建設業界の若手技術者が抱える「膨大な書類作成業務」と「経験不足による判断迷い」を劇的に解消するために開発された、次世代型の施工管理サポートSaaSです。

AIエージェントとの高度なペアプログラミングにより、堅牢なインフラ設計から緻密な帳票出力ロジックまで、エンタープライズの現場で即座に実運用可能な品質水準を妥協なく実現しました。

* **ターゲット:** 建設現場の若手エンジニアおよび施工管理者
* **コアバリュー:** A3縦1枚に最適化された日次・週次・月次報告書の自動生成、および若手技術者向けの技術基準即時検索機能（RAG）

## 技術スタック

モダンかつスケーラブルな構成を採用し、AIの推論能力とシステムの可用性を最大限に引き出しています。

* **Frontend:** Vite + React, Tailwind CSS (レスポンシブデザイン / Z-layout)
* **Backend:** Node.js, Express, Cloud Run
* **Database / Auth:** Supabase (PostgreSQL / pgvector)
* **AI Engine:** Google Gemini 1.5 Flash / 2.0 Pro (Hybrid Inference)
* **Infrastructure:** Firebase Hosting (Frontend), Google Cloud Run (Backend)

## 技術的ハイライト (Technical Moats)

### 1. 職人技レベルのA3縦1枚フルカラー帳票生成
ブラウザ側（jspdf + html2canvas）で、日次・週次・月次の報告書を「A3縦サイズ1枚」に1ミリのズレもなく正確にレンダリングし収める独自ロジックを実装。印刷時のスケーリング最適化（既定100%でのフィット）により、そのまま発注者や所長へ提出可能なクオリティを実現しました。

### 2. 若手向け技術検索機能（ハルシネーション・ゼロのRAG）
200万文字を超えるJASS（日本建築学会標準仕様書）や法令データをセマンティック検索。AIが「それらしい嘘」をつくのを防ぎ、必ず引用元を提示するインテリジェントな技術検索機能を搭載。経験の浅い若手技術者が、現場で迷った際に正しい技術的根拠へ即座にアクセスできる環境を提供します。

### 3. ハイブリッドAI推論によるコスト・精度の最適化
* **通常日報:** 高速なGemini Flashでリアルタイム処理。
* **週次・月次の俯瞰分析:** 高度な推論が可能なGemini Proを採用。
文脈の重要度に応じてモデルを自律的に使い分けることで、エンタープライズ要件に求められるUXとコストパフォーマンスを両立しました。

### 4. 堅牢なインフラとセキュリティ設計
Cloud Runのメモリ最適化（ストリーミングパースによるOOM回避）や、厳格なContent Security Policy (CSP) 設定、アクセス解析導入など、リリース初日からエンタープライズ水準の安全な運用基盤を構築しています。

## セットアップ方法 (Getting Started)

本プロジェクトをローカル環境で起動・カスタマイズする手順は以下の通りです。

**1. リポジトリのクローン**
```bash
git clone https://github.com/gorillasys-00/qcdse-ai-showcase.git
cd qcdse-ai-showcase
```

**2. 依存関係のインストール**
プロジェクトは `frontend` と `backend` の2つのディレクトリで構成されています。
```bash
# Frontend
cd frontend
npm install

# Backend
cd ../backend
npm install
```

**3. 環境変数 (.env) の設定**
`backend` ディレクトリ直下に `.env` ファイルを作成し、以下のキーを設定してください。
```text
# Supabase
SUPABASE_URL=your_supabase_url
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

# Gemini API
GEMINI_API_KEY=your_gemini_api_key

# Stripe (決済機能を使用する場合)
STRIPE_SECRET_KEY=your_stripe_secret_key
STRIPE_WEBHOOK_SECRET=your_webhook_secret

# その他
PORT=5000
```

**4. 開発サーバーの起動**
```bash
# Backend (Port 5000)
cd backend
npm start

# Frontend (Vite)
cd frontend
npm run dev
```

## 搭載ナレッジベース (Knowledge Base)

本システムのRAGエンジンには、日本の建設業界でデファクトスタンダードとなっている技術文献が事前に構造化・ベクトル化されて搭載されています。

* **JASS（日本建築学会 建築工事標準仕様書）:** 建築工事の品質を左右する技術基準。
* **公共建築工事標準仕様書:** 国土交通省が定める施工マニュアル。
* **社内施工指針サンプル:** 現場ごとの特殊ルールを模したダミーデータ。

**ハルシネーション防止メカニズム:** 検索された文脈（Context）に該当する情報がない場合、AIは「回答不可」を明示するようにプロンプトレベルで厳格に統制されています。また、回答には必ず `[引用: JASS 5 節 2.3]` のようにソースを付与し、人間の技術者が最終確認できる信頼性を担保しています。

## ナレッジの拡張・追加方法 (How to Expand Knowledge)

企業独自の「現場ルールブック」や「過去の事故事例」を追加し、AIを自社専用にチューニングすることが可能です。

**データ追加のフロー**
1.  **データ準備:** PDF、Excel、またはXML形式の技術資料を用意。
2.  **パースと分割:** バックエンド内のスクリプトでテキストを抽出し、AIのコンテキスト制限に合わせた適切なチャンクサイズに分割。
3.  **ベクトル化:** テキストをベクトル形式に変換（エンベディング）。
4.  **保存:** Supabaseの `pgvector` エクステンションを有効にしたテーブルに格納。
5.  **推論:** ユーザーの質問に対し、コサイン類似度検索で関連チャンクを抽出し、推論モデルへ注入。

## システムのカスタマイズ (Customization)

**帳票フォーマットの調整**
A3 PDFのレイアウトはフロントエンドのCSSで厳密に制御されています。建設現場の慣習に合わせた微調整が可能です。
* **プリントスケール:** CSSの `@media print` セクションにて、A3縦サイズ1枚に情報を完璧に収めるためのスケーリング設定を適用済み。

**新しい帳票タイプの追加**
「安全パトロール記録」や「品質検査記録」などの新帳票を追加する場合：
1.  バックエンドに新しいエンドポイントを作成。
2.  システムプロンプト（System Instruction）を新規帳票用にチューニング。
3.  フロントエンドのUIコンポーネントを追加。

## エンタープライズ品質を担保する開発アプローチ

本プロジェクトは、開発スピードのみならず、本番稼働に耐えうる「品質」と「可用性」を最優先に設計・実装を行いました。AIエージェントを高度に活用し、以下のフェーズを経て強固なシステムを構築しています。

* **フェーズ1: 堅牢なインフラ基盤構築**
    Supabaseによる認証・データベース設計と、Cloud Runへのデプロイ設計。
* **フェーズ2: コアロジックの精度向上**
    A3縦出力エンジンのミリ単位のレイアウト調整と、Gemini Flash/Proのハイブリッド推論の最適化。
* **フェーズ3: RAGエンジンの高度化**
    JASS等の技術基準データのベクトル化と、ハルシネーションを完全に防ぐプロンプトの徹底。
* **フェーズ4: 実運用に向けた最適化**
    OOM（メモリ不足エラー）対策、CSPの厳格化、Stripe決済連携、およびアクセス解析の導入。
